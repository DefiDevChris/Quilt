# Accessibility Features

QuiltCorgi is designed to be accessible to quilters of all technical skill levels and abilities. This guide covers all accessibility features, keyboard shortcuts, and interaction modes available in the design studio.

## Universal Design Principles

QuiltCorgi follows WCAG AAA accessibility standards:

- **Touch Targets**: All interactive elements meet the 44×44px minimum size requirement
- **High Contrast**: Text and UI elements maintain sufficient contrast ratios
- **Clear Labels**: All tools and actions have explicit text labels, not just icons
- **Forgiving Interactions**: Multiple ways to accomplish the same task
- **Persistent Undo/Redo**: Always-visible undo and redo buttons for easy error recovery

## Interaction Modes

### Drag-and-Drop Mode (Traditional)

The classic way to add blocks and fabrics to your canvas:

1. **Blocks**: Click and drag a block from the library onto the canvas
2. **Fabrics**: Click and drag a fabric swatch onto a patch to fill it
3. **Visual Feedback**: Items become semi-transparent while dragging, cursor changes to indicate valid drop zones

### Tap-to-Place Mode (Accessible Alternative)

A more forgiving interaction mode that doesn't require precise dragging:

#### Placing Blocks
1. **Tap to Select**: Click once on a block in the library to select it
2. **Visual Confirmation**: The selected block highlights with a blue border and ring
3. **Floating Indicator**: A banner appears at the top of the screen: "Tap canvas to place [Block Name]"
4. **Tap to Place**: Click anywhere on the canvas to place the block at that location
5. **Grid Snapping**: If grid snap is enabled, the block automatically aligns to the nearest grid intersection
6. **Repeat or Cancel**: The block stays selected so you can place multiple copies, or click the X button to cancel

#### Filling with Fabrics
1. **Tap to Select**: Click once on a fabric swatch in the library to select it
2. **Visual Confirmation**: The selected fabric highlights with a blue border and ring
3. **Floating Indicator**: A banner appears: "Tap a patch to fill with [Fabric Name]"
4. **Tap to Fill**: Click on any patch on the canvas to apply the fabric pattern
5. **Multi-Fill**: The fabric stays selected so you can fill multiple patches without re-selecting
6. **Cancel**: Click the X button in the floating indicator to deselect

## Keyboard Shortcuts

### Canvas Navigation
- **Space + Drag**: Pan around the canvas
- **Ctrl + Scroll**: Zoom in/out
- **Ctrl + 0**: Reset zoom to 100%

### Editing
- **Ctrl + Z**: Undo last action
- **Ctrl + Shift + Z** or **Ctrl + Y**: Redo
- **Ctrl + C**: Copy selected object(s)
- **Ctrl + V**: Paste copied object(s)
- **Ctrl + X**: Cut selected object(s)
- **Delete** or **Backspace**: Delete selected object(s)
- **Ctrl + A**: Select all objects on canvas
- **Ctrl + D**: Duplicate selected object(s)

### Tools
- **V**: Select tool
- **R**: Rectangle tool
- **C**: Circle tool
- **L**: Line tool
- **T**: Text tool
- **Esc**: Cancel current tool or selection

### Project
- **Ctrl + S**: Save project
- **Ctrl + E**: Export image

## Undo/Redo Overlay

The undo and redo buttons are always visible at the top-center of the canvas:

- **Large Touch Targets**: 48×48px buttons for easy clicking
- **Visual State**: Buttons are disabled (grayed out) when no actions are available
- **Keyboard Hints**: Hover tooltips show keyboard shortcuts
- **Text Labels**: Clear "Undo" and "Redo" labels beneath icons

## Screen Reader Support

All interactive elements include proper ARIA labels:

- **Buttons**: `aria-label` describes the action (e.g., "Undo last action")
- **Tools**: `aria-pressed` indicates active tool state
- **Panels**: `aria-expanded` indicates open/closed state
- **Status Messages**: `aria-live="polite"` announces important state changes

## Quick Start Workflows

The dashboard features three large, accessible entry points:

1. **Photo to Pattern** (Pro feature): Upload a quilt photo and extract blocks with AI
2. **Start from Template**: Browse pre-made patterns
3. **Blank Project**: Start with an empty canvas

Each card:
- Meets 44×44px minimum touch target
- Has clear icon + title + description
- Provides visual feedback on hover
- Works with keyboard navigation (Tab to focus, Enter to activate)

## Color and Contrast

- **Text Contrast**: All text meets WCAG AAA standards (7:1 for normal text, 4.5:1 for large text)
- **Focus Indicators**: Keyboard focus is clearly visible with blue rings
- **Color Independence**: Information is never conveyed by color alone

## Responsive Design

- **Desktop Studio**: Full design tools with multi-worktable support
- **Mobile Redirect**: Mobile users are redirected to the dashboard (studio requires desktop)
- **Flexible Layouts**: UI adapts to different screen sizes and zoom levels

## Getting Help

- **Help Panel**: Click the "?" icon in the studio for contextual help
- **Tooltips**: Hover over any tool or button for a description
- **Keyboard Shortcuts**: Press **Ctrl + /** to view all shortcuts (if implemented)
