# API Specification
**Project:** QuiltCorgi
**Version:** 1.0
**Date:** March 26, 2026
**Purpose:** Every API endpoint, request/response schema, error codes, and data flow.

---

## Base Configuration
- **Base URL:** `/api` (Next.js App Router route handlers)
- **Authentication:** NextAuth.js session-based JWT. Protected routes check session via `getServerSession()`.
- **Content Type:** `application/json`
- **API Versioning:** None (internal API only, no public consumers)

---

## Authentication Endpoints

Authentication is handled entirely by NextAuth.js. The following routes are auto-generated by the NextAuth.js App Router handler at `/app/api/auth/[...nextauth]/route.ts`:

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/auth/signin` | Sign-in page (redirects to providers) |
| POST | `/api/auth/signin/[provider]` | Initiate OAuth flow for provider (google, facebook, apple, twitter) |
| POST | `/api/auth/signin/credentials` | Email/password sign-in |
| GET | `/api/auth/callback/[provider]` | OAuth callback handler |
| POST | `/api/auth/signout` | Sign out (invalidate session) |
| GET | `/api/auth/session` | Get current session data |
| GET | `/api/auth/providers` | List configured auth providers |
| GET | `/api/auth/csrf` | Get CSRF token |

These endpoints follow the NextAuth.js specification and are not custom-built. → See [07-AUTH-SECURITY.md § Authentication Flow] for the complete auth lifecycle.

## Resource Endpoints

### Projects

#### GET /api/projects
**Description:** List the authenticated user's projects.
**Auth:** Required — Free, Pro, Admin
**Rate Limit:** None

**Request:**
| Parameter | Location | Type | Required | Description |
|-----------|----------|------|----------|-------------|
| page | Query | Integer | No | Page number (default: 1) |
| limit | Query | Integer | No | Items per page (default: 20, max: 50) |
| sort | Query | String | No | Sort field: `updatedAt` (default), `createdAt`, `name` |
| order | Query | String | No | Sort order: `desc` (default), `asc` |

**Success Response:** `200 OK`
```json
{
  "projects": [
    {
      "id": "uuid",
      "name": "My First Quilt",
      "description": null,
      "thumbnailUrl": "https://cdn.quiltcorgi.com/thumbnails/uuid.png",
      "unitSystem": "imperial",
      "isPublic": false,
      "lastSavedAt": "2026-03-26T12:00:00Z",
      "createdAt": "2026-03-20T08:30:00Z",
      "updatedAt": "2026-03-26T12:00:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 5,
    "totalPages": 1
  }
}
```

**Error Responses:**
| Status | Code | Message | When |
|--------|------|---------|------|
| 401 | UNAUTHORIZED | "Authentication required" | No valid session |

#### POST /api/projects
**Description:** Create a new project.
**Auth:** Required — Free (max 3 projects), Pro, Admin

**Request Body:**
```json
{
  "name": "string — project name (max 255 chars)",
  "unitSystem": "imperial | metric",
  "canvasWidth": "number — canvas width in units",
  "canvasHeight": "number — canvas height in units",
  "gridSettings": {
    "enabled": "boolean",
    "size": "number — grid interval in units",
    "snapToGrid": "boolean"
  }
}
```

**Success Response:** `201 Created`
```json
{
  "id": "uuid",
  "name": "My Quilt",
  "unitSystem": "imperial",
  "canvasWidth": 48.0,
  "canvasHeight": 48.0,
  "canvasData": {},
  "gridSettings": { "enabled": true, "size": 1, "snapToGrid": true },
  "thumbnailUrl": null,
  "isPublic": false,
  "lastSavedAt": "2026-03-26T12:00:00Z",
  "createdAt": "2026-03-26T12:00:00Z",
  "updatedAt": "2026-03-26T12:00:00Z"
}
```

**Error Responses:**
| Status | Code | Message | When |
|--------|------|---------|------|
| 401 | UNAUTHORIZED | "Authentication required" | No valid session |
| 403 | PROJECT_LIMIT_REACHED | "Free plan allows up to 3 projects. Upgrade to Pro for unlimited projects." | Free user has 3+ projects |
| 422 | VALIDATION_ERROR | "Canvas dimensions must be between 1 and 200." | Invalid input |

### Blocks

#### GET /api/blocks
**Description:** List blocks from the library with search and filtering.
**Auth:** Required — Free (limited to 100 system blocks), Pro (all blocks), Admin (all blocks)

**Request:**
| Parameter | Location | Type | Required | Description |
|-----------|----------|------|----------|-------------|
| search | Query | String | No | Fuzzy search by name and tags |
| category | Query | String | No | Filter by category |
| scope | Query | String | No | `system` (default library), `user` (user's custom blocks), `all` |
| page | Query | Integer | No | Page number (default: 1) |
| limit | Query | Integer | No | Items per page (default: 50, max: 100) |

**Success Response:** `200 OK`
```json
{
  "blocks": [
    {
      "id": "uuid",
      "name": "Log Cabin",
      "category": "Traditional",
      "subcategory": "Cabin",
      "tags": ["beginner", "strips", "traditional"],
      "thumbnailUrl": "https://cdn.quiltcorgi.com/blocks/log-cabin.png",
      "isDefault": true,
      "isLocked": false
    }
  ],
  "pagination": { "page": 1, "limit": 50, "total": 6000, "totalPages": 120 }
}
```

Note: `isLocked` is computed server-side. For free users, system blocks beyond the first 100 have `isLocked: true`. `svgData` and `fabricJsData` are NOT included in list responses — only fetched on individual block GET.

---

#### GET /api/blocks/[id]
**Description:** Get a single block with full vector data for canvas insertion.
**Auth:** Required — owner (for custom blocks) or any user (for system blocks within tier limits)

**Success Response:** `200 OK`
```json
{
  "id": "uuid",
  "name": "Log Cabin",
  "category": "Traditional",
  "svgData": "<svg>...</svg>",
  "fabricJsData": { "...Fabric.js object JSON..." },
  "tags": ["beginner", "strips"],
  "isDefault": true
}
```

**Error Responses:**
| Status | Code | Message | When |
|--------|------|---------|------|
| 401 | UNAUTHORIZED | "Authentication required" | No valid session |
| 403 | PRO_REQUIRED | "This block requires a Pro subscription." | Free user accessing locked block |
| 404 | NOT_FOUND | "Block not found." | Invalid block ID |

---

#### POST /api/blocks
**Description:** Save a new custom block.
**Auth:** Required — Pro, Admin

**Request Body:**
```json
{
  "name": "string — block name (max 255 chars)",
  "category": "string — category name",
  "svgData": "string — SVG path data",
  "fabricJsData": "object — Fabric.js serialized object",
  "tags": ["string array — searchable tags"]
}
```

**Success Response:** `201 Created` — returns the created block object.

---

#### DELETE /api/blocks/[id]
**Description:** Delete a custom block (user-created only).
**Auth:** Required — owner only

**Success Response:** `204 No Content`

**Error Responses:**
| Status | Code | Message | When |
|--------|------|---------|------|
| 403 | FORBIDDEN | "System blocks cannot be deleted." | Attempting to delete a default block |

**Auth:** None required

**Request:**

| sort | Query | String | No | `newest` (default), `popular` |

### Stripe Webhooks

#### POST /api/stripe/webhook
**Description:** Stripe webhook handler. Verifies webhook signature and processes events.
**Auth:** Stripe webhook signature verification (no user session required).

**Handled Events:**
| Event | Action |
|-------|--------|
| `checkout.session.completed` | Create/update subscription record, set user role to `pro`, create `subscription_activated` notification |
| `customer.subscription.updated` | Update subscription record (plan, status, period dates) |
| `customer.subscription.deleted` | Set subscription status to `canceled`, schedule role revert to `free` at period end |
| `invoice.payment_failed` | Set subscription status to `past_due`, create notification |

**Success Response:** `200 OK` — `{ "received": true }`

## Error Code Reference

| Status | Code | Description |
|--------|------|-------------|
| 400 | BAD_REQUEST | Malformed request body or parameters |
| 401 | UNAUTHORIZED | No valid session / authentication required |
| 403 | FORBIDDEN | User lacks permission for this resource |
| 403 | PRO_REQUIRED | Feature requires Pro subscription |
| 403 | PROJECT_LIMIT_REACHED | Free plan project limit exceeded |
| 404 | NOT_FOUND | Resource does not exist |
| 409 | CONFLICT | Resource already exists (e.g., duplicate like, duplicate community post) |
| 422 | VALIDATION_ERROR | Input validation failed |
| 429 | RATE_LIMITED | Too many requests |
| 500 | INTERNAL_ERROR | Unexpected server error |

## Pagination

All list endpoints use offset-based pagination:
```
GET /api/[resource]?page=2&limit=20
```

Response includes:
```json
{
  "pagination": {
    "page": 2,
    "limit": 20,
    "total": 150,
    "totalPages": 8
  }
}
```

## Filtering & Sorting

Standard query parameter patterns:
- `?sort=fieldName` — sort by field
- `?order=asc|desc` — sort direction
- `?search=query` — fuzzy text search
- `?[filterName]=value` — exact match filter
